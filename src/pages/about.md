我从工业现场出发，正在把 AI 工具链变成自己的交付能力。

在工厂里摸爬滚打过多年：调过 PLC、接过大电阻、跟供应商扯过皮、也被客户催过工期。见过太多"能用就行"的代码和"明天就要上线"的需求，慢慢练出了一套**把模糊需求快速做成可运行系统**的能力。

现在把这条能力迁移到 AI 时代——用 Codex、Cursor 这些工具，把想法变成代码，把代码变成产品。

## 擅长什么

**从需求到交付的完整闭环**

- 工厂环境练出来的**现场意识**：知道设备怎么跑、节拍怎么算、异常怎么处理
- C# / .NET 上位机开发：能写工业级交互界面，也能对接各种协议（Modbus、OPC UA 等）
- AI Coding 工具链：重度使用 Codex、Cursor，配合多模型 API 做自动化工作流

**代码要能跑在真实环境里**

见过太多"能跑在开发机上，生产环境一堆问题"的系统。所以写代码时会多想一步：异常怎么处理、日志怎么打、参数怎么配。

## 关注方向

AI Coding / Industrial Software / Automation Workflow / C# / .NET / Project Delivery
